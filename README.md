# Morning Story Ranker

Internal dashboard that surfaces the top external gaming stories from the last 24 hours, ranked transparently by source authority × recency + Reddit engagement.

See `morning-story-ranker-prd.md` for the full spec.

## Stack

- **Next.js 16** (App Router, no `src/`) + React 19 + TypeScript + Tailwind 4
- **Postgres** via [Neon](https://neon.tech) (serverless) — `postgres` driver
- **Magic-link auth** via [Resend](https://resend.com), HMAC-signed session cookies (no DB session store)
- **Ingestion**: `rss-parser` for outlets, public Reddit JSON for subs (no API key)
- **Deployment target**: Render Web Service + Render Cron Job at `09:00 UTC` daily

The auth pattern (HMAC cookies, magic links) mirrors the converter apps under `Documents/` — see `CLAUDE.md` at the repo root.

## Local development

Requires Node 20+, a Neon project, a Resend account with a verified sender domain.

```powershell
cd Documents/morning-story-ranker
cp .env.example .env.local
# Fill in DATABASE_URL, AUTH_SECRET, RESEND_API_KEY, MAIL_FROM, CRON_TOKEN
# Optional: INITIAL_ALLOWED_EMAILS=you@yourcompany.com
npm install
npm run db:init      # runs migrations + seed + seeds allowed_emails
npm run dev          # http://localhost:3000
```

`npm run db:init` is idempotent — re-running it skips already-applied migrations, re-runs the seed (the seed itself is idempotent), and upserts `INITIAL_ALLOWED_EMAILS` into the allowlist.

If you'd rather seed manually, the equivalent SQL is:

```sql
insert into allowed_emails (email) values ('you@yourcompany.com');
```

## Deploy to Render

1. **Push this directory to its own Git repo** (GitHub etc.). The converter apps under `Documents/` are each their own repo — same pattern.
2. **Provision Neon** at [neon.tech](https://neon.tech). Copy the **pooled** connection string.
3. **Provision Resend** at [resend.com](https://resend.com). Verify a sender domain (add the DNS records they tell you to add) and create an API key.
4. **Create the Render Blueprint**: in the Render dashboard → New → Blueprint → select this repo. Render reads `render.yaml` and provisions:
   - `morning-story-ranker` — the web service
   - `morning-story-ranker-cron` — the daily 09:00 UTC trigger
5. **Fill the `sync: false` env vars** when Render prompts you:

   | Var | Value |
   |---|---|
   | `DATABASE_URL` | Neon pooled connection string |
   | `RESEND_API_KEY` | Resend API key |
   | `MAIL_FROM` | `Morning Story Ranker <noreply@yourverifieddomain.com>` |
   | `INITIAL_ALLOWED_EMAILS` | Your email (comma-separate for teammates). Seeded on every deploy |
   | `PUBLIC_BASE_URL` | Leave blank unless you put a custom domain in front of Render. Code falls back to `RENDER_EXTERNAL_URL` |

   `AUTH_SECRET`, `CRON_TOKEN`, `CRON_TARGET_URL` are handled automatically:
   - `AUTH_SECRET` and `CRON_TOKEN` are generated by Render at create time.
   - The cron service reads `CRON_TOKEN` and `CRON_TARGET_URL` from the web service via `fromService` — no manual copying.

6. **First deploy runs migrations** automatically via `preDeployCommand: npm run db:init`. Watch the deploy log to confirm `init-db: ok`.
7. Visit your service URL → request a magic link from `/login` → click the link → you're in.
8. To trigger the ranking job manually before 09:00 UTC tomorrow: click **Refresh** on the dashboard.

## Required environment variables

| Var | Purpose |
|---|---|
| `DATABASE_URL` | Neon pooled connection string |
| `AUTH_SECRET` | 16+ char random for HMAC cookies. `openssl rand -hex 32` |
| `RESEND_API_KEY` | Resend API key for magic-link emails |
| `MAIL_FROM` | Verified sender, e.g. `Morning Story Ranker <noreply@yourdomain.com>` |
| `CRON_TOKEN` | Bearer token for `/api/cron/rank`. 8+ chars. `openssl rand -hex 32` |
| `PUBLIC_BASE_URL` (optional) | Override origin used in email links (set in prod) |

## Auth flow

1. User submits email at `/login`
2. `POST /api/auth/magic-link` looks up `allowed_emails`. If absent → silent no-op. If present → create a single-use 15-minute token (sha256-hashed in DB), email the verify URL via Resend, redirect to `/login?sent=1`
3. User clicks the email link → `GET /api/auth/verify?token=…&next=…`
4. Token is consumed (atomic `UPDATE … WHERE used_at IS NULL RETURNING`), session cookie issued (30-day HMAC), redirect to `next` or `/`
5. `proxy.ts` enforces the gate on every subsequent request

Rotating `AUTH_SECRET` invalidates every existing session.

## Triggering the ranking job

After running migrations + seed + adding yourself to `allowed_emails`:

```powershell
# Local one-shot — admin force bypasses caches
curl -X POST "http://localhost:3000/api/cron/rank?vertical=gaming&force=1" `
  -H "Authorization: Bearer $env:CRON_TOKEN"

# In production, Render Cron does this for you at 09:00 UTC daily
```

The `Refresh` button on the dashboard does the same thing for logged-in users (with a 60s server-side cooldown, no token required since it's session-gated).
